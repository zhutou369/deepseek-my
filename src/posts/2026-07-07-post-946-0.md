### Akta Perlindungan Data Peribadi (PDPA) 2010: Versi & Garis Panduan

PDPA 2010 ni sebenarnya digubal untuk kawal selia macam mana data peribadi diproses dalam urusan komersial. Ia ada tujuh prinsip utama yang wajib kita patuhi: Prinsip Am, Prinsip Notis dan Pilihan, Prinsip Penzahiran, Prinsip Keselamatan, Prinsip Penyimpanan, Prinsip Integriti Data, dan Prinsip Akses. Kalau anda pembangun atau pemilik PKS yang guna DeepSeek API, anda ni dikira sebagai "pengguna data" dan ada tanggungjawab di bawah akta ni. Garis Panduan Penggunaan DeepSeek API (Versi 1.2, Januari 2024) pula, walaupun dari DeepSeek sendiri, boleh kita jadikan rujukan untuk amalan terbaik dalam menguruskan input dan output API.

### 3 Langkah Utama Pematuhan PDPA untuk Pengguna DeepSeek API

Nampak macam rumit je nak patuhi PDPA ni, tapi bila kita guna DeepSeek API, kita boleh pecahkan kepada langkah-langkah yang senang nak faham.

#### 1. Peringkat Perancangan & Pengumpulan Data: Tanya Diri Dahulu

Sebelum hantar apa-apa data ke DeepSeek API, cuba berhenti sejenak dan tanya: "Eh, aku perlukan ke data peribadi ni?" Ramai yang terlepas pandang bahagian penting ni. Untuk elak masalah di kemudian hari, anda patut:

a. **Minimumkan Pengumpulan:** Kumpul data peribadi yang betul-betul perlu sahaja. Kalau DeepSeek API boleh berfungsi dengan data yang dah di-anonimkan (dibuang identiti peribadi) atau disamarkan (pseudonymized), guna je kaedah tu. Contohnya, jika anda nak analisis sentimen komen pelanggan, tak perlu pun nama penuh mereka.

b. **Dapatkan Kebenaran:** Yang ni memang penting! Kalau data peribadi tak dapat dielakkan, anda wajib dapatkan kebenaran yang jelas dari individu pemilik data tersebut. Beritahu mereka kenapa data mereka diperlukan, bagaimana ia akan diproses (iaitu, melalui DeepSeek API), dan untuk tujuan apa. Pastikan kebenaran itu boleh ditarik balik bila-bila masa.

c. **Notis Privasi yang Jelas:** Sediakan notis privasi yang senang difaham di laman web atau aplikasi anda. Terangkan secara telus tentang jenis data yang dikumpul, kenapa nak guna, dan dengan siapa data itu mungkin dikongsi (DeepSeek sebagai pemproses data anda). Ini adalah sebahagian daripada prinsip notis di bawah PDPA.

#### 2. Peringkat Pemprosesan & Penyimpanan Data: Jaga Data Elok-Elok

Bila data anda mula diproses oleh DeepSeek API, dan juga masa ia disimpan, ada beberapa perkara yang perlu kita titikberatkan betul-betul:

a. **Pilih Model yang Sesuai:** DeepSeek ada banyak model. Ada yang sesuai untuk "fine-tuning" dengan data kita, dan ada yang untuk penggunaan "prompt" sekali lalu. Fahami cara kerja model yang anda guna. Adakah DeepSeek akan simpan data yang anda hantar untuk latihan model mereka? Baca terma perkhidmatan mereka. Lazimnya, platform API terkemuka tidak menggunakan data pengguna akhir untuk melatih model tanpa kebenaran.

b. **Keselamatan Data:** Ini tanggungjawab anda. Walaupun DeepSeek ada langkah keselamatan mereka sendiri, anda perlu pastikan data yang anda hantar ke API dan terima dari API itu selamat. Guna sambungan yang secure (HTTPS), pastikan [kunci API anda](/posts/deepseek-api-key-and-limits/) tak terdedah kat orang lain, dan lindungi sistem anda dari pencerobohan. Ini selaras dengan prinsip keselamatan PDPA.

c. **Pengendalian Data Sensitif:** Untuk data yang sangat sensitif (contohnya: maklumat kesihatan, kepercayaan agama), adalah lebih baik elak dari menghantarnya ke DeepSeek API atau mana-mana API awam lain. Kalau terpaksa sangat, pertimbangkan penyelesaian AI yang boleh dijalankan di [pelayan mandiri seperti Ollama](/posts/deepseek-ollama-local-setup/) di premis anda, di mana anda ada kawalan penuh ke atas data dan infrastruktur.

#### 3. Peringkat Penggunaan & Penamatan Data: Ada Batasan dan Tempohnya

Selepas data diproses dan digunakan, ada juga prinsip PDPA yang perlu kita patuhi.

a. **Tujuan Penggunaan:** Guna data hanya untuk tujuan yang anda dah maklumkan dan dapatkan kebenaran dari pemilik data. Jangan sesekali guna data yang dihantar ke DeepSeek API untuk tujuan lain yang tak diberitahu.

b. **Tempoh Penyimpanan:** PDPA menetapkan anda tak boleh simpan data peribadi lebih lama daripada yang diperlukan untuk capai tujuan ia dikumpulkan. Jika data anda dah diproses oleh DeepSeek API dan hasilnya sudah digunakan, anda patut fikirkan untuk padam data asal yang tak diperlukan lagi. Pastikan juga anda faham macam mana DeepSeek uruskan penyimpanan data input anda, dan jika ada pilihan untuk padamnya.

c. **Hak Akses & Pembetulan:** Individu ada hak untuk minta akses kepada data peribadi mereka dan minta pembetulan kalau ada kesilapan. Sistem anda perlu ada cara untuk uruskan permintaan ni, dan ini mungkin melibatkan anda dapatkan data dari DeepSeek API jika ia masih disimpan di sana, atau pastikan ia dah dipadam.

### Soalan Lazim (FAQ)

#### 1. Apa itu Data Peribadi di bawah PDPA?

Secara ringkas, data peribadi adalah sebarang maklumat yang berkaitan dengan seseorang individu, dari mana identiti individu itu boleh dikenal pasti secara langsung atau tak langsung. Ini termasuk nama, nombor kad pengenalan, alamat, nombor telefon, alamat e-mel, cap jari, dan kadang-kadang juga alamat IP atau data lokasi kalau ia boleh dikaitkan dengan individu tertentu. Maklumat sensitif macam data kesihatan, agama, dan pandangan politik ada perlindungan tambahan.

#### 2. Adakah DeepSeek API patuh PDPA?

DeepSeek sebagai penyedia perkhidmatan API biasanya ada polisi privasi mereka sendiri yang mematuhi undang-undang perlindungan data antarabangsa macam GDPR. Namun begitu, sebagai pengguna data (anda), anda tetap bertanggungjawab pastikan cara anda *guna* DeepSeek API dan data yang anda hantar kepadanya adalah patuh PDPA Malaysia. DeepSeek adalah pemproses data bagi pihak anda, tapi tanggungjawab utama terletak pada anda untuk dapatkan persetujuan dan pastikan data itu diuruskan dengan betul.

#### 3. Apa jadi kalau tak patuh PDPA?

Jangan main-main! Kegagalan mematuhi PDPA boleh membawa kepada denda yang berat (sampai RM500,000) dan/atau hukuman penjara. Lepas tu, reputasi perniagaan atau projek anda boleh terjejas teruk, dan anda mungkin hilang kepercayaan pelanggan. Untuk PKS, ini boleh jadi malapetaka besar.

#### 4. Boleh ke guna DeepSeek API untuk data sensitif?

Nasihat terbaik saya adalah, elakkanlah. Kalau betul-betul terpaksa sangat, anda mesti ada langkah keselamatan yang super ketat, dapatkan kebenaran yang jelas dan bertulis dari pemilik data, dan pastikan data itu dienkripsi. Namun, risiko tetap tinggi. Lebih baik cari penyelesaian tempatan atau model yang boleh dihoskan sendiri untuk elakkan pendedahan data sensitif kepada pihak ketiga. Dalam konteks memuat naik data sensitif untuk pembelajaran model melalui [DeepSeek API](/posts/deepseek-prompt-basics/), ia perlukan pertimbangan yang sangat mendalam dari segi etika dan perundangan.

DeepSeek API adalah perkhidmatan awan, di mana anda hantar data anda ke pelayan DeepSeek untuk diproses. Ini bermakna data anda merentasi rangkaian dan diproses di luar kawalan fizikal anda. [Ollama](/posts/deepseek-ollama-local-setup/) pula membolehkan anda jalankan model bahasa besar (termasuk model yang setara dengan DeepSeek) secara lokal di komputer atau pelayan mandiri anda. Dengan Ollama, data anda tak keluar dari rangkaian anda, bagi kawalan penuh ke atas privasi dan keselamatan data.
