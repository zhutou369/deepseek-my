Dunia digital sentiasa bergerak pantas, dan bagi Perusahaan Kecil dan Sederhana (PKS) di Malaysia, keupayaan untuk menghasilkan kandungan berkualiti secara konsisten adalah kunci kepada transformasi digital. Pengurusan dan penerbitan blog secara manual sering kali memakan masa serta sumber yang berharga, sekali gus menghalang PKS daripada memberi tumpuan kepada inovasi teras. Pendekatan ini bukan sahaja menjimatkan masa tetapi juga memastikan konsistensi dalam penerbitan kandungan, membolehkan PKS kekal relevan dalam pasaran yang kompetitif.

## Mengapa Automasi Penting untuk PKS di Malaysia?

Dalam lanskap perniagaan Malaysia yang dinamik, PKS berhadapan dengan pelbagai cabaran, daripada persaingan sengit hingga kekangan sumber. Automasi memainkan peranan penting dalam mengatasi cabaran ini dengan menawarkan pelbagai kelebihan:

### Peningkatan Kecekapan Operasi
Dengan mengautomasikan tugas-tugas berulang seperti penjanaan draf artikel blog, PKS dapat mengalihkan sumber manusia mereka untuk memberi tumpuan kepada aktiviti yang lebih strategik seperti pembangunan produk, perhubungan pelanggan, atau inovasi. Ini membolehkan syarikat beroperasi dengan lebih cekap dan produktif.

### Konsistensi Kandungan dan Penjenamaan
Automasi memastikan kandungan baharu diterbitkan secara berkala, menjaga kesinambungan blog anda. Dengan menggunakan arahan (prompts) yang sama atau serupa kepada DeepSeek, anda boleh mengekalkan nada suara dan gaya penulisan yang konsisten, mengukuhkan jenama anda di mata audiens.

### Skalabiliti dan Pertumbuhan
Apabila perniagaan anda berkembang, keperluan untuk kandungan juga akan meningkat. Sistem automasi yang dibina dengan GitHub Actions dan DeepSeek boleh diskalakan dengan mudah untuk menghasilkan lebih banyak kandungan mengikut keperluan, tanpa memerlukan penambahan tenaga kerja yang besar. Ini adalah aset berharga untuk PKS yang ingin mengembangkan rangkaian dan jangkauan digital mereka.

### Pengoptimuman Sumber
PKS sering beroperasi dengan bajet yang terhad. Melabur dalam automasi seperti ini, walaupun memerlukan sedikit usaha awal, akan menjimatkan kos operasi jangka panjang yang berkaitan dengan pengambilan penulis kandungan atau agensi pemasaran. Ia membolehkan penggunaan sumber yang lebih optimum.

Melalui automasi, PKS di Malaysia boleh melonjakkan strategi pemasaran kandungan mereka, membuka laluan kepada transformasi digital yang lebih menyeluruh dan mencapai pertumbuhan yang mampan.

## Memahami Komponen Utama Automasi

Untuk membina sistem automasi ini, kita akan menggunakan tiga teknologi utama yang saling melengkapi.

### GitHub Actions
GitHub Actions adalah platform integrasi berterusan (CI/CD) dan automasi yang membolehkan anda mengautomasikan, menyesuaikan, dan melaksanakan alir kerja (workflows) pembangunan anda secara terus dalam repositori GitHub. Ia boleh mencetuskan peristiwa berdasarkan tindakan di repositori anda (seperti `push` atau `pull request`), atau pada jadual masa yang ditetapkan. Bagi kes penggunaan kita, GitHub Actions akan menjadi "otak" yang mengkoordinasikan proses penjanaan dan penerbitan blog.

### DeepSeek
DeepSeek adalah salah satu model AI generasi teks yang canggih, direka untuk memahami dan menjana teks bahasa semula jadi yang berkualiti tinggi. Melalui API DeepSeek, kita boleh menghantar arahan (prompts) dan menerima kembali artikel blog yang ditulis secara automatik. Keupayaan DeepSeek untuk menghasilkan kandungan yang relevan dan koheren menjadikannya pilihan ideal untuk penjanaan draf artikel. Anda perlu mendaftar untuk mendapatkan kunci API daripada DeepSeek untuk berinteraksi dengannya.

### 11ty (Eleventy)
11ty adalah penjana tapak statik (static site generator) yang ringkas dan fleksibel. Ia mengambil data dan templat (templates) anda, kemudian mengubahnya menjadi fail HTML statik yang boleh dihantar terus ke pelayar web. 11ty sangat sesuai untuk blog kerana kecekapannya, keselamatan, dan kelajuan yang tinggi. Dalam sistem automasi ini, DeepSeek akan menjana kandungan dalam format Markdown, yang kemudiannya akan digunakan oleh 11ty untuk membina laman blog.

## Langkah Demi Langkah: Menyiapkan Automasi Posting Blog

Sekarang, mari kita mulakan proses penyediaan automasi posting blog anda.

### Mendapatkan Kunci API DeepSeek

Perkara pertama ialah mendapatkan kunci API DeepSeek anda. Lawati laman web DeepSeek AI, daftar atau log masuk, dan navigasi ke bahagian yang berkaitan dengan API atau kunci akses. Salin kunci API anda dan pastikan anda menyimpannya dengan selamat. Jangan sesekali kongsikan kunci ini secara terbuka atau masukkan terus dalam kod yang akan di-`commit` ke repositori.

### Menyimpan Kunci API dengan Selamat di GitHub Secrets

Untuk memastikan kunci API anda selamat, kita akan menyimpannya sebagai "GitHub Secret".

1.  Di repositori GitHub anda, pergi ke **Settings**.
2.  Di menu sebelah kiri, pilih **Secrets and variables > Actions**.
3.  Klik pada butang **New repository secret**.
4.  Untuk **Name**, masukkan `DEEPSEEK_API_KEY`.
5.  Untuk **Secret**, tampal kunci API DeepSeek anda.
6.  Klik **Add secret**.

Sekarang, kunci API anda boleh diakses oleh GitHub Actions anda tanpa mendedahkannya dalam fail konfigurasi.

### Mencipta Workflow GitHub Actions

Workflow GitHub Actions anda akan mendefinisikan langkah-langkah automasi. Anda perlu mencipta direktori `.github/workflows/` di dalam repositori anda, dan kemudian fail YAML di dalamnya (cth: `generate-blog-post.yml`).

```yaml
# .github/workflows/generate-blog-post.yml

name: Automasi Blog DeepSeek & 11ty

on:
  workflow_dispatch: # Membolehkan pencetus manual dari GitHub Actions UI
  # schedule:
  #   - cron: '0 0 * * *' # Boleh dicetuskan setiap hari pada tengah malam (UTC)

jobs:
  generate-post:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout kod repositori
      uses: actions/checkout@v4

    - name: Sediakan Node.js
      uses: actions/setup-node@v4
      with:
        node-version: '20' # Gunakan versi Node.js yang sesuai untuk 11ty anda

    - name: Pasang kebergantungan 11ty
      run: npm install

    - name: Sediakan Python untuk skrip AI
      uses: actions/setup-python@v5
      with:
        python-version: '3.x'

    - name: Pasang kebergantungan Python
      run: pip install requests markdownify

    - name: Jana artikel blog dengan DeepSeek
      env:
        DEEPSEEK_API_KEY: ${{ secrets.DEEPSEEK_API_KEY }}
      run: python .github/workflows/generate_post.py

    - name: Commit dan hantar artikel baharu
      run: |
        git config user.name "GitHub Actions Bot"
        git config user.email "actions@github.com"
        git add posts/*.md # Sesuaikan dengan folder posting 11ty anda
        git commit -m "feat: Menambah artikel blog baharu yang dijana AI" || echo "Tiada perubahan untuk di-commit"
        git push
```

#### Penjelasan Workflow YAML:

*   **`name`**: Nama paparan untuk workflow anda.
*   **`on`**: Menentukan bila workflow ini akan berjalan.
    *   `workflow_dispatch`: Membolehkan anda mencetuskan workflow ini secara manual dari tab Actions di GitHub.
    *   `schedule`: (Dikomen buat masa ini) Anda boleh menjadualkan workflow untuk berjalan pada selang masa tertentu menggunakan sintaks cron.
*   **`jobs`**: Blok ini mengandungi satu atau lebih kerja yang akan dilaksanakan.
    *   `generate-post`: Nama kerja ini.
    *   `runs-on: ubuntu-latest`: Menentukan bahawa kerja akan berjalan pada persekitaran Ubuntu yang terkini.
    *   **`steps`**: Senarai tindakan yang akan dilaksanakan secara berurutan.
        *   `Checkout kod repositori`: Mengambil kod repositori anda.
        *   `Sediakan Node.js` & `Pasang kebergantungan 11ty`: Menyiapkan persekitaran Node.js dan memasang kebergantungan yang diperlukan oleh projek 11ty anda.
        *   `Sediakan Python` & `Pasang kebergantungan Python`: Menyiapkan persekitaran Python dan memasang pustaka `requests` (untuk API DeepSeek) dan `markdownify` (untuk menukar HTML kepada Markdown, jika DeepSeek menghasilkan HTML).
        *   `Jana artikel blog dengan DeepSeek`: Langkah ini akan memanggil skrip Python yang bertanggungjawab untuk berinteraksi dengan DeepSeek. `DEEPSEEK_API_KEY` diakses daripada GitHub Secrets.
        *   `Commit dan hantar artikel baharu`: Selepas skrip Python menjana fail Markdown, langkah ini akan menambahkannya ke Git, membuat commit, dan menghantarnya ke repositori anda. Ini penting supaya 11ty dapat mengesan artikel baharu pada build seterusnya.

*GitHub Actions menyediakan platform automasi yang fleksibel untuk pembangunan perisian dan kandungan.*

### Skrip Python untuk Interaksi DeepSeek

Cipta fail Python baharu di dalam folder `.github/workflows/` yang sama, dinamakan `generate_post.py`. Skrip ini akan menghantar arahan ke DeepSeek, menerima respons, dan menyimpannya sebagai fail Markdown yang sesuai untuk 11ty.

```python
# .github/workflows/generate_post.py
import os
import requests
import json
import datetime
from markdownify import markdownify as md

# Konfigurasi API DeepSeek
DEEPSEEK_API_KEY = os.environ.get('DEEPSEEK_API_KEY')
DEEPSEEK_API_URL = "https://api.deepseek.com/chat/completions" # Pastikan ini URL API yang betul

HEADERS = {
    "Authorization": f"Bearer {DEEPSEEK_API_KEY}",
    "Content-Type": "application/json"
}

def generate_content(prompt):
    payload = {
        "model": "deepseek-chat", # Atau model DeepSeek lain yang anda ingin gunakan
        "messages": [
            {"role": "system", "content": "Anda adalah pakar penulisan blog yang pakar dalam topik teknologi dan perniagaan untuk PKS Malaysia. Anda akan menulis artikel dalam Bahasa Malaysia."},
            {"role": "user", "content": prompt}
        ],
        "temperature": 0.7,
        "max_tokens": 1500, # Hadkan panjang respons untuk mengawal kos dan saiz
    }
    try:
        response = requests.post(DEEPSEEK_API_URL, headers=HEADERS, data=json.dumps(payload))
        response.raise_for_status() # Lontarkan pengecualian untuk kod status HTTP yang tidak berjaya
        return response.json()['choices'][0]['message']['content']
    except requests.exceptions.RequestException as e:
        print(f"Ralat semasa memanggil API DeepSeek: {e}")
        return None

def create_markdown_file(title, content):
    # Bersihkan tajuk untuk nama fail yang sah
    file_slug = title.lower().replace(" ", "-").replace(":", "").replace("/", "").replace(".", "")[:50]
    file_name = f"posts/{datetime.date.today().isoformat()}-{file_slug}.md" # Sesuaikan dengan struktur folder 11ty anda
    
    # Bina Front Matter untuk 11ty
    front_matter = f"""---
title: "{title}"
date: {datetime.datetime.now().isoformat()}
tags: ["automasi", "ai", "deepseek", "11ty"]
layout: "layouts/post.njk" # Sesuaikan dengan layout 11ty anda
description: "{content[:150].split('.')[0]}..." # Ambil 150 aksara pertama sebagai deskripsi ringkas
---

"""
    # Gabungkan front matter dan kandungan
    full_content = front_matter + content

    os.makedirs(os.path.dirname(file_name), exist_ok=True)
    with open(file_name, "w", encoding="utf-8") as f:
        f.write(full_content)
    print(f"Fail Markdown '{file_name}' berjaya dicipta.")

if __name__ == "__main__":
    # Ini adalah contoh prompt. Anda boleh menjana prompt secara dinamik
    # atau menyimpannya dalam fail berasingan.
    blog_prompt = "Tulis artikel blog bertajuk 'Manfaat Transformasi Digital untuk PKS di Malaysia' fokus kepada penggunaan teknologi AI dan automasi. Artikel mestilah dalam Bahasa Malaysia, 800-1000 patah perkataan, dengan pengenalan, 3-4 isi utama dan kesimpulan."
    
    # Dapatkan tajuk dari prompt atau hasilkan secara automatik
    # Untuk demo ini, kita ambil terus dari prompt.
    # Dalam sistem yang lebih canggih, anda mungkin minta AI menjana tajuk juga.
    article_title = "Manfaat Transformasi Digital untuk PKS di Malaysia" 

    print("Menghubungi DeepSeek untuk menjana kandungan...")
    generated_content = generate_content(blog_prompt)

    if generated_content:
        # DeepSeek biasanya akan mengembalikan teks biasa atau Markdown.
        # Jika ia mengembalikan HTML, anda boleh menukarkannya kepada Markdown:
        # markdown_content = md(generated_content)
        markdown_content = generated_content # Anggap DeepSeek kembalikan Markdown
        
        create_markdown_file(article_title, markdown_content)
    else:
        print("Gagal menjana kandungan.")

```

#### Penjelasan Skrip Python:

*   **Konfigurasi API**: Menetapkan kunci API dan URL DeepSeek.
*   **`generate_content(prompt)`**: Fungsi ini menghantar arahan ke API DeepSeek dan mengembalikan kandungan yang dijana.
    *   `model`: Anda boleh menukar model DeepSeek (`deepseek-chat` atau yang lain) di sini.
    *   `messages`: Mendefinisikan peranan `system` (memberi konteks kepada AI) dan `user` (arahan sebenar).
    *   `temperature`: Mengawal kreativiti respons (nilai lebih tinggi = lebih kreatif).
    *   `max_tokens`: Menghadkan panjang respons untuk mengelakkan penjanaan yang terlalu panjang dan mengawal kos.
*   **`create_markdown_file(title, content)`**: Fungsi ini mengambil tajuk dan kandungan yang dijana, membina "Front Matter" yang diperlukan oleh 11ty (tajuk, tarikh, tag, layout), dan menyimpan keseluruhan kandungan ke dalam fail Markdown di dalam folder `posts/` anda.
*   **Main Block (`if __name__ == "__main__":`)**:
    *   Menentukan `blog_prompt` yang akan dihantar ke DeepSeek. Anda boleh mengubah prompt ini untuk menjana topik yang berbeza.
    *   Memanggil `generate_content` untuk mendapatkan artikel.
    *   Memanggil `create_markdown_file` untuk menyimpan artikel yang dijana.

### Mencetuskan Workflow Automasi Anda

Setelah anda `commit` kedua-dua fail `.github/workflows/generate-blog-post.yml` dan `.github/workflows/generate_post.py` ke repositori GitHub anda, pergi ke tab **Actions** di repositori anda. Anda akan melihat workflow "Automasi Blog DeepSeek & 11ty".

Klik pada workflow tersebut, kemudian klik pada butang **Run workflow** di sebelah kanan. Ini akan mencetuskan workflow secara manual. GitHub Actions akan mula berjalan, melaksanakan setiap langkah.

Selepas workflow berjaya, anda akan melihat fail Markdown baharu di dalam folder `posts/` repositori anda. Ini akan mencetuskan proses `build` 11ty anda (jika anda mempunyai workflow CI/CD berasingan untuk `deploy` 11ty), atau anda perlu membina semula tapak web 11ty anda secara manual untuk melihat artikel baharu itu.

## Mengoptimumkan Prompt untuk DeepSeek

Kualiti kandungan yang dijana oleh DeepSeek sangat bergantung pada kualiti arahan (prompt) yang anda berikan. Berikut adalah beberapa tips:

*   **Bersikap Spesifik**: Nyatakan topik, tujuan, audiens, nada, dan format yang anda inginkan.
*   **Sertakan Kata Kunci**: Jika ada kata kunci SEO yang perlu disertakan, sebutkannya.
*   **Nyatakan Panjang**: Contoh: "800-1000 patah perkataan."
*   **Struktur yang Jelas**: Minta struktur tertentu (pengenalan, 3 perenggan isi, kesimpulan).
*   **Bahasa**: Pastikan anda menyatakan "dalam Bahasa Malaysia".
*   **Peranan AI**: Berikan DeepSeek peranan, contohnya: "Anda adalah pakar pemasaran digital..."

Contoh prompt yang lebih baik:
`"Tulis artikel blog komprehensif untuk PKS di Malaysia bertajuk 'Strategi Pemasaran Digital Berkesan Pasca Pandemik'. Fokus pada pentingnya penggunaan media sosial, SEO tempatan, dan iklan dalam talian. Sertakan contoh relevan untuk pasaran Malaysia dan gunakan bahasa formal tetapi mudah difahami. Panjang antara 900-1100 patah perkataan, dengan pendahuluan, tiga subtopik utama, dan kesimpulan. Pastikan kandungan mesra SEO untuk carian 'Pemasaran Digital PKS Malaysia'."`

## Cabaran dan Pertimbangan

Walaupun automasi ini menawarkan banyak manfaat, terdapat beberapa cabaran dan pertimbangan yang perlu diambil kira:

*   **Kos API**: Penggunaan API DeepSeek akan melibatkan kos. Pantau penggunaan anda untuk mengelakkan bil yang tidak dijangka.
*   **Kualiti Kandungan**: Walaupun DeepSeek sangat baik, ia mungkin memerlukan semakan manusia untuk mencapai kualiti yang setara dengan penulis profesional. Pengeditan dan penyesuaian mungkin masih diperlukan.
*   **Kepelbagaian Topik**: Jika anda ingin menjana pelbagai topik, anda perlu menguruskan prompt yang berbeza atau mencipta sistem untuk menjana prompt secara dinamik.
*   **Etika AI**: Pastikan kandungan yang dijana tidak melanggar etika atau hak cipta.
